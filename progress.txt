- [x] BUG: Skill check rolls broken on remote server
  - Root cause: `svelte-range-slider-pips` binds `value` as `number[]`, but `selectedRollMode` in `CheckRollDialog.svelte` was initialised as a scalar.
    Before the slider hydrates (slower on remote due to network latency), `selectedRollMode[0]` returned `undefined`, producing an invalid roll formula and silently failing.
    Attacks (via `ItemActivationConfigDialog`) were unaffected because they pass `selectedRollMode` directly without indexing.
  - Fix: initialise `selectedRollMode` as `[scalar]` array and update the `$derived` `getRollFormula` call to use `selectedRollMode[0]`.
  - File: `src/view/dialogs/CheckRollDialog.svelte`
  - Commit: FIX: Skill check rolls broken on remote server due to RangeSlider array binding

- [x] BUG: Chat cards not appearing when sidebar is collapsed
  - Root cause: `renderChatMessageHTML` hook stored Svelte component on `message._svelteComponent`.
    When FoundryVTT re-rendered the same message for the collapsed-sidebar notification overlay,
    the hook unmounted the component from the main chat log element before mounting to the notification.
    On sidebar expand, users saw blank (un-Sveltified) chat cards.
  - Fix: track component on the target DOM element (`target._nimbleSvelteComponent`) instead of
    on the message, so each rendered copy manages its own lifecycle independently.
  - File: `src/hooks/renderChatMessage.ts`
  - Commit: FIX: Chat cards not appearing when sidebar is collapsed
